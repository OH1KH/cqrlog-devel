CQRLOG for LINUX by OK7AN & OK1RR

Start and stop scripts

"Run at cqrlog start" and "Run at cqrlog stop" calls to scripts:
~/.config/cqrlog/start.sh
~/.config/cqrlog/stop.sh

They are called when program is up/goes down and rigctld is still running. You can do anything with them. At start you can switch on your coffee maker and when cqlog is closed switch coffee maker off :D :D :D
At stop phase 1000ms (1sec) wait time is added to let script access to rigctld before it goes down. This should be enough to execute a command.
For powering rig via rigctld power command make start.sh like:

     ---------------------------------------------
     #!/bin/bash
     echo -n -e '\x87'1 |  nc localhost 4532
     exit
     ---------------------------------------------
For stop.sh script copy start.sh and change 1 (after 87') to 0 to get rigctld set rig power off.
Or change 1 (after 87') to 2 to get rigctld set rig to stand by.
Remember to chmod a+x start.sh (and stop.sh) to make them execute for all.

NOTE: Your rig may not start with rigctld power-on command if it is totally powered of (CPU is not running).
In that case try to set it to stand by-state at closing time. Then it may start again with power-on command.
